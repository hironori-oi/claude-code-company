# コミュニケーションプロトコル

## 1. 指示・報告の流れ

### トップダウン（指示）
```
オーナー → CEO → 各部署
```
- CEOが指示を適切な部署に分解・委任する
- 各部署は自分の権限範囲内で実行する
- 権限外の判断が必要な場合はCEOにエスカレーションする

### ボトムアップ（報告）
```
各部署 → CEO → オーナーへのサマリー
```
- 各部署はCEOに報告する（オーナーに直接報告しない）
- CEOが各部署の報告を統合・要約してオーナーに報告する
- 緊急時のみ、CEOの判断でオーナーに即時報告する

## 2. 横連携ルール

### 許可される横連携
| 連携元 | 連携先 | 目的 |
|--------|--------|------|
| 開発 | リサーチ | 技術的な実現性確認、ライブラリ調査依頼 |
| PM | 開発 | 工数見積もり、タスクの技術的確認 |
| マーケ | リサーチ | 市場データの共有、競合情報の取得 |
| マーケ | 開発 | 技術的な提案内容の確認 |
| Web運営 | 開発 | HP開発の技術的支援 |
| Web運営 | マーケ | ブランドガイドライン、コンテンツ方針の確認 |
| レビュー | 開発 | レビュー指摘の技術的説明 |

### 横連携のルール
- 横連携の結果は必ずCEOにも報告する
- 横連携で意見が対立した場合はCEOが裁定する
- 横連携は情報共有と相談に限る（指示権限はCEOのみ）

## 3. 報告フォーマット

### 定期報告
各部署は業務完了時に以下を含む報告をCEOに行う:
1. 実施内容のサマリー
2. 成果物のファイルパス
3. 発見した課題・リスク
4. 推奨する次のアクション

### 緊急報告
以下の場合はただちにCEOに報告:
- セキュリティに関わる問題の発見
- スケジュールに大きな影響を与える問題
- クライアント要件と実装の重大な矛盾
- 技術的に実現不可能な要件の発見

## 4. 意思決定の記録
- 全ての重要な意思決定は `projects/{案件ID}/decisions.md` に記録する
- 記録フォーマット:
  ```
  ## YYYY-MM-DD: {決定事項}
  - 決定者: {CEO/部署名}
  - 背景: {なぜこの判断が必要だったか}
  - 選択肢: {検討した選択肢}
  - 決定内容: {選んだ選択肢}
  - 理由: {選択の根拠}
  ```
