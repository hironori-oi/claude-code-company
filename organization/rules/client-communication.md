# クライアントコミュニケーション規約

## 基本方針
- メールを主要なコミュニケーション手段とする
- 進捗報告は **2週間に1回** の頻度で実施する
- 丁寧かつ簡潔な文面を心がける
- 技術用語は平易な言葉に置き換えて説明する

## 進捗報告

### 頻度
- 定期報告: 2週間に1回（隔週）
- 臨時報告: 重要な問題・変更がある場合は随時

### 報告タイミング
- 各案件の開始時にクライアントと報告日を合意する
- 報告日は原則として金曜日とする（週末前に状況を共有）

### 進捗報告メールテンプレート
```
件名: 【進捗報告】{案件名} - {日付}

{クライアント名} 様

いつもお世話になっております。
{案件名}の進捗状況をご報告いたします。

■ 全体進捗: {XX}%

■ 今期間の実施内容
・{実施内容1}
・{実施内容2}

■ 次回までの予定
・{予定内容1}
・{予定内容2}

■ 確認事項（ある場合）
・{クライアントに確認したい事項}

■ 備考
・{その他伝えるべき事項}

ご不明な点がございましたら、お気軽にご連絡ください。
よろしくお願いいたします。
```

## コミュニケーションの段階別テンプレート

### 問い合わせ受領時
```
件名: 【受領確認】お問い合わせありがとうございます

{クライアント名} 様

お問い合わせいただきありがとうございます。
内容を確認させていただきました。

詳細をお伺いしたく、以下についてお教えいただけますでしょうか。

1. {ヒアリング項目1}
2. {ヒアリング項目2}
3. {ヒアリング項目3}

ご回答いただき次第、お見積り・ご提案書を作成いたします。
よろしくお願いいたします。
```

### 提案書送付時
```
件名: 【ご提案】{案件名}について

{クライアント名} 様

先日のお打ち合わせ内容をもとに、ご提案書を作成いたしました。
添付ファイルをご確認ください。

3つのプランをご用意しておりますので、
ご予算・ご要望に合わせてご検討いただければ幸いです。

ご質問やご要望がございましたら、お気軽にお申し付けください。
よろしくお願いいたします。
```

### 納品時
```
件名: 【納品】{案件名} - 納品のご報告

{クライアント名} 様

{案件名}の開発が完了いたしましたので、納品させていただきます。

■ 納品物
・本番環境URL: {URL}
・管理画面URL: {URL}
・ソースコード: {GitHub URL}
・操作マニュアル: 添付

■ アカウント情報
・別途メールにてお送りいたします

■ 保証期間
・納品日より{XX}日間、不具合の修正に対応いたします

ご確認のほど、よろしくお願いいたします。
```

## メール対応ルール
- 問い合わせには **24時間以内** に一次返信する（内容確認中でも受領連絡）
- 返信が遅れる場合は、いつまでに回答できるか目安を伝える
- 重要な合意事項はメールで文書化する（口頭のみにしない）
- 見積もり・提案書はPDFで送付する
