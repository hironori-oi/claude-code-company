# テスト方針

## 基本方針
- テストは Claude Code に委任しつつ、品質を担保する
- 案件の規模・性質に応じてテストレベルを調整する
- 最低限のテストは全案件で必須とする

## テストレベル定義

### Level 1: 最低限（小規模案件・LP等）
- リンター（ESLint）の通過
- TypeScript の型チェック通過
- 主要画面の手動動作確認

### Level 2: 標準（通常のWebアプリ案件）
- Level 1 の全項目
- 主要なAPI・Server Actions のユニットテスト
- 認証フローの動作テスト
- フォームバリデーションのテスト

### Level 3: 充実（大規模・重要案件）
- Level 2 の全項目
- E2Eテスト（Playwright）：主要ユーザーフローをカバー
- パフォーマンステスト（Lighthouse CI）
- セキュリティスキャン

## テストツール
| ツール | 用途 |
|--------|------|
| **Vitest** | ユニットテスト・統合テスト |
| **Playwright** | E2Eテスト |
| **ESLint** | 静的解析 |
| **TypeScript** | 型チェック |

## テスト実行タイミング
- コミット前: ESLint + TypeScript 型チェック
- PR作成時: 全テスト実行
- デプロイ前: 全テスト通過を確認

## レビュー部門との連携
- レビュー部門はテストカバレッジと品質を確認する
- Critical な機能にテストがない場合は差し戻し対象
