# デザインガイドライン

## デザイン原則

### 1. AI感を出さないクリーンなデザイン
- ランダムなグラデーション、グラスモーフィズムの乱用を避ける
- 角丸のサイズを統一する（shadcn/ui のデフォルトに従う）
- ボーダーとスペーシングで構造を表現する
- 意味のないアニメーション・エフェクトは使わない

### 2. ミニマルだが温かみのあるデザイン
- 余白を十分に取る
- 情報密度を適切に保つ（詰め込みすぎない）
- タイポグラフィで階層を表現する
- 写真やイラストを適切に使い、機械的になりすぎない

### 3. モバイルファースト
- モバイル表示を最初に設計する
- タッチターゲットは最低 44x44px
- モバイルでの操作性を最優先する

## カラーシステム

### ベースカラー
- **ニュートラル**: zinc 系（slate や neutral も可）
- **アクセント**: 案件ごとにクライアントのブランドカラーに合わせる
- **アクセントカラーは1色**: 多色使いは避ける

### ダーク/ライトモード
- 全案件でダーク/ライトモード切り替えを実装する
- `next-themes` でテーマ管理
- デフォルト: `system`（OS設定に追従）
- CSS変数ベースでカラーを管理（shadcn/ui の標準方式）

### コントラスト
- テキストと背景のコントラスト比: WCAG AA（4.5:1）以上
- 大きいテキスト（18px以上）: 3:1 以上

## タイポグラフィ

### フォント
- **本文**: Geist Sans（`next/font` で読み込み）
- **コード・数値**: Geist Mono
- **日本語**: Noto Sans JP または Geist Sans のフォールバック

### サイズ階層
| 用途 | サイズ | ウェイト |
|------|--------|---------|
| h1 | 2.25rem (36px) | Bold (700) |
| h2 | 1.875rem (30px) | Semibold (600) |
| h3 | 1.5rem (24px) | Semibold (600) |
| h4 | 1.25rem (20px) | Medium (500) |
| 本文 | 1rem (16px) | Regular (400) |
| 補足 | 0.875rem (14px) | Regular (400) |
| キャプション | 0.75rem (12px) | Regular (400) |

## アイコン・絵文字

### 絵文字の使用禁止
- UIに絵文字（Unicode Emoji）を使用しない
- ボタン、ラベル、見出し、ナビゲーション等すべてのUI要素で絵文字禁止
- 装飾目的の絵文字も使用しない

### アイコンライブラリ
- **Heroicons** (https://heroicons.com/) を標準アイコンとして使用する
- `npm install @heroicons/react` でインストール
- Outline（24px）と Solid（20px）の2スタイルが利用可能
- 用途に応じて使い分ける:
  - **Outline**: ナビゲーション、ツールバー、一般的なUI要素
  - **Solid**: アクティブ状態、強調表示、小さいサイズ（16px以下）
- サイズは `className="h-5 w-5"` のように Tailwind で指定する
- アイコン単体のボタンには必ず `aria-label` を設定する（アクセシビリティ）

### 使用例
```tsx
import { MagnifyingGlassIcon } from '@heroicons/react/24/outline'
import { HomeIcon } from '@heroicons/react/24/solid'

// ナビゲーションアイコン（Outline）
<MagnifyingGlassIcon className="h-5 w-5" />

// アクティブ状態（Solid）
<HomeIcon className="h-5 w-5" />
```

## コンポーネント設計

### shadcn/ui の利用方針
- 標準コンポーネントを最大限活用する
- カスタムコンポーネントは shadcn/ui の設計パターンに準拠する
- `cn()` ユーティリティを使ったスタイル合成

### よく使うコンポーネント構成
| 画面タイプ | 推奨構成 |
|-----------|---------|
| ダッシュボード | Card + Table + Filters |
| 設定画面 | Tabs + Card + Form |
| 一覧画面 | DataTable + Pagination |
| 詳細画面 | Card + Separator + Tabs |
| フォーム | Form + Input + Select + Button |
| モーダル | Dialog + Form |
| ナビゲーション | Sheet（モバイル） + NavigationMenu（PC） |
| 確認ダイアログ | AlertDialog |
| 通知 | Toast (Sonner) |

### 状態表示
- ローディング: Skeleton コンポーネントでレイアウトを維持
- エラー: Alert コンポーネントで表示
- 空状態: イラスト + メッセージ + CTAボタン
- これらの状態を必ず設計に含める（放置しない）

## レスポンシブブレークポイント
| ブレークポイント | Tailwind | 用途 |
|----------------|----------|------|
| < 640px | デフォルト | モバイル |
| ≥ 640px | `sm:` | 大きいモバイル |
| ≥ 768px | `md:` | タブレット |
| ≥ 1024px | `lg:` | デスクトップ |
| ≥ 1280px | `xl:` | 大画面 |

## 画像・メディア
- `next/image` を必ず使用する（自動最適化）
- WebP / AVIF フォーマットで配信（Vercel で自動変換）
- Lazy loading をデフォルトにする（ファーストビュー画像を除く）
- alt テキストを必ず設定する（アクセシビリティ）

## アニメーション
- 過度なアニメーションは使わない
- トランジション: 150-300ms、ease-out が基本
- Framer Motion は必要な場合のみ（ページ遷移、複雑なインタラクション）
- `prefers-reduced-motion` を尊重する
