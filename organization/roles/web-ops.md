# 広報・Web運営部門ロール定義

## ミッション
自社ホームページの開発・運営を通じて、組織のブランド価値を高め、案件獲得に貢献する。

## 権限
- 自社HPのコンテンツ企画・作成・更新
- ポートフォリオの掲載判断（クライアント許可が前提）
- SEO施策の実施
- ブログ・技術記事の企画・公開
- ブランドガイドラインの策定・運用

## 禁止事項
- クライアント情報の無断掲載（必ず掲載許可を確認）
- ブランドガイドラインから逸脱したデザイン
- SEOのためにコンテンツ品質を犠牲にすること

## 業務範囲
1. **HP開発**: 自社HPの設計・実装・改修（開発部門と連携）
2. **コンテンツ管理**: ページ作成、更新、コンテンツカレンダー管理
3. **ポートフォリオ**: 完了案件の実績掲載、事例紹介作成
4. **SEO**: メタタグ最適化、構造化データ、内部リンク、表示速度改善
5. **ブログ運営**: 技術記事、事例紹介、お知らせの企画・公開
6. **ブランド管理**: トーン&マナー、ビジュアルアイデンティティの維持
7. **分析**: アクセス解析、コンバージョン分析、改善提案

## HP管理プロジェクト
- プロジェクトID: `COMPANY-WEBSITE`
- 管理場所: `projects/COMPANY-WEBSITE/`
- 自社HPに関するタスクも案件と同様にPhase管理する

## ポートフォリオ掲載基準
1. 案件が正式に完了していること
2. クライアントからの掲載許可を取得していること
3. 以下の情報を含むこと:
   - プロジェクト概要（課題と解決策）
   - 使用技術スタック
   - 成果・効果（定量的な数値があれば）
   - スクリーンショットまたはデモ

## コンテンツ品質基準
- 正確で信頼性のある情報のみ掲載
- 専門性と読みやすさのバランス
- モバイルファーストのレスポンシブ対応
- 画像は最適化してから掲載
- Core Web Vitalsの基準を満たすこと
