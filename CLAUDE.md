# claude-code-company - AI組織運営システム

## 概要

このプロジェクトは、Claude Codeのスラッシュコマンドとエージェント機能を活用したAI組織運営システムです。
ユーザー（オーナー）がCEOに指示を出し、各部署のAIエージェントが専門的な視点で業務を遂行します。

## 組織構造

```
オーナー（ユーザー）
  └── CEO（/ceo）- 最高意思決定・全体統括
        ├── 秘書（/secretary）- 案件受付・記録・スケジュール管理
        ├── PM（/pm）- プロジェクト管理・タスク分解・進捗管理
        ├── リサーチ（/research）- 技術調査・競合分析・実現性検証
        ├── 開発（/dev）- 設計・実装・テスト・デプロイ
        ├── マーケティング（/marketing）- 提案書・市場分析・成長戦略
        ├── レビュー（/review）- 品質管理・コードレビュー・最終検収
        └── 広報Web運営（/web-ops）- 自社HP開発・運営・ポートフォリオ管理
```

## 基本ルール

### 1. 指示・報告の流れ
- **トップダウン**: オーナー → CEO → 各部署
- **ボトムアップ**: 各部署 → CEO → オーナーへサマリー
- 各部署が直接オーナーに報告することは禁止（必ずCEOを経由）
- 横連携の結果もCEOに報告すること

### 2. 成果物ルール
- 全ての成果物は `projects/{案件ID}/` 配下に保存する
- **アプリケーション実体（ソースコード）は `projects/{案件ID}/app/` 配下に配置する** — Desktop 直下や workspace ルートにアプリを作らない。外部実体を取り込む場合は物理移動・submodule・junction のいずれかで `app/` 配下からアクセス可能にする（詳細: `organization/rules/project-setup-checklist.md` 最優先ルール）
- 各部署のレポートは `projects/{案件ID}/reports/` に保存する
- テンプレートに従ったフォーマットで出力する（`organization/templates/`参照）
- 意思決定は必ず理由とともに `projects/{案件ID}/decisions.md` に記録する

### 3. 品質基準
- Phase移行には品質ゲートの通過が必須（`organization/rules/quality-gates.md`参照）
- コードは必ずレビュー部門のチェックを経てから納品する
- セキュリティ・アクセシビリティは妥協しない

### 4. 案件管理
- 案件IDは `PRJ-XXX` 形式で連番管理する
- 全案件は `dashboard/active-projects.md` で一元管理する
- 案件のPhase変更時は必ず `dashboard/active-projects.md` を更新する

### 5. 自社HP（Web運営）
- 自社HPの開発・運営は `projects/COMPANY-WEBSITE/` で管理する
- 完了した案件の実績は広報Web運営部門がポートフォリオとして掲載する
- HP関連のタスクも案件と同様にPhase管理する

### 6. ナレッジ蓄積
- 案件完了後はKPT（Keep/Problem/Try）振り返りを実施する
- 再利用可能な知見は `organization/knowledge/` に蓄積する
- 技術選定の判断基準や、よくある落とし穴を記録する

## コマンド一覧

| コマンド | 用途 |
|---------|------|
| `/ceo {指示}` | CEOとして全体統括・部署への指示 |
| `/new-project {概要}` | 新規案件の登録 |
| `/secretary {指示}` | 秘書業務 |
| `/pm {指示}` | プロジェクト管理 |
| `/research {指示}` | 技術リサーチ |
| `/dev {指示}` | 開発業務 |
| `/marketing {指示}` | マーケティング業務 |
| `/review {指示}` | 品質レビュー |
| `/web-ops {指示}` | 自社HP開発・運営 |
| `/status {案件ID?}` | ステータス確認（ID省略で全案件） |
| `/report {種類}` | 経営レポート生成 |

## 事業方針

- **ターゲット**: 中小企業向けWebアプリ開発
