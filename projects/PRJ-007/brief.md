## 案件概要

### 背景・課題
PRJ-002はWebブラウザ上でPDF帳票からデータを自動抽出するシステムだが、Phase A〜DでサーバーサイドAPI（`/api/v1/*`）を実装した。このAPIを活用し、ユーザーのWindows PC上で「フォルダにPDFを入れるだけで自動処理」を実現するデスクトップクライアントを開発する。

PRJ-006（PRJ-005カミレス向けデスクトップクライアント）は別プロダクト（PRJ-005）向けに設計されているため、PRJ-002用には専用クライアントを新規開発する。

### 目的・ゴール
PRJ-002のAPI連携機能を活用し、ユーザーのWindows PC上でフォルダ監視 → PDF自動送信 → データ抽出 → CSV出力を一気通貫で行うデスクトップアプリを開発する。

### ターゲットユーザー
- PRJ-002を利用する事務担当者
- 複合機でスキャンしたPDFをPC上のフォルダに保存 → 手作業でWebアプリにアップロードしている担当者

### 製品コンセプト
「フォルダにPDFを入れるだけ。あとは全部自動。」

## PRJ-002 API仕様（接続先）

| エンドポイント | 機能 |
|---------------|------|
| `GET /api/v1/templates` | テンプレート一覧取得 |
| `POST /api/v1/extract` | 単一PDF同期抽出 |
| `POST /api/v1/batch` | 複数PDFバッチ抽出（最大20件） |
| `GET /api/v1/jobs/:id` | ジョブステータス確認 |
| `GET /api/v1/jobs/:id/results?format=csv` | 結果CSV取得 |
| `POST /api/v1/webhooks` | Webhook登録 |

- **認証**: `X-API-Key` ヘッダー（`pk_` プレフィックス）
- **仕様書**: `projects/PRJ-002/api-docs/openapi.yaml`

## 主要機能

### 1. API接続設定
- PRJ-002のAPI URL入力
- APIキー入力・検証（`GET /api/v1/templates` で疎通確認）
- 認証状態のセキュアストレージ保存（OS資格情報マネージャー）

### 2. テンプレート選択
- PRJ-002からテンプレート一覧を取得・表示
- テンプレート選択・プロファイル保存

### 3. フォルダ設定
- 入力フォルダ（監視対象ディレクトリ選択）
- 出力フォルダ（CSV出力先ディレクトリ選択）
- 処理済みフォルダ（抽出完了PDF移動先ディレクトリ選択）

### 4. フォルダ監視
- 入力フォルダの新規PDFファイルを自動検知
- ファイル書き込み完了確認（サイズ安定チェック）
- ポーリング間隔設定

### 5. 自動抽出
- 単一ファイル: `POST /api/v1/extract` で即時処理
- 複数ファイル蓄積時: `POST /api/v1/batch` でバッチ処理
- 処理中のプログレス表示

### 6. CSV出力
- `GET /api/v1/jobs/:id/results?format=csv` でCSV取得
- 出力フォルダにCSVファイルとして保存
- エンコーディング設定（UTF-8 BOM / Shift-JIS）

### 7. ファイル移動
- 処理完了PDFを処理済みフォルダに自動移動
- エラー時は入力フォルダに残留

### 8. システムトレイ常駐
- バックグラウンド動作
- トレイアイコンから状態確認・設定画面呼び出し

### 9. 処理ログ
- 処理履歴の一覧表示（日時、ファイル名、ステータス、結果サマリー）

### 10. Windows通知
- 処理完了・エラー時のトースト通知
