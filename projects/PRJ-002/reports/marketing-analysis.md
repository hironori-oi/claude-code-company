## 1. 市場分析

### 1.1 AI-OCR / インテリジェントOCR市場の現状

#### グローバル市場
- **市場規模（2025年推定）**: 約150億ドル（OCR全体）、うちAI-OCR/インテリジェントOCRは約50-60億ドル
- **成長率（CAGR）**: 2024-2030年で約15-20%の年間成長率
- **主要ドライバー**:
  - 企業のDX（デジタルトランスフォーメーション）推進
  - クラウドベースOCRの普及
  - AI/ML技術の飛躍的進歩（特にLLM・VLMの登場）
  - ペーパーレス化の法規制強化（電子帳簿保存法等）

#### 日本国内市場
- **市場規模（2025年推定）**: 約500-700億円（AI-OCR関連）
- **成長率**: 年率20-25%（国内DX需要の高まりにより世界平均を上回る）
- **背景**:
  - 2024年1月の改正電子帳簿保存法の完全義務化
  - インボイス制度への対応需要
  - 中小企業のDX遅延（大企業との格差拡大）
  - 労働力不足に伴う業務自動化の必要性

### 1.2 中小企業のペーパーレス化・DXの課題

| 課題カテゴリ | 具体的な課題 | 影響度 |
|-------------|------------|--------|
| **コスト** | 既存AI-OCRサービスの高額な月額費用（月10-50万円） | 極めて高い |
| **IT人材不足** | 導入・運用を担うIT人材がいない | 高い |
| **既存業務との統合** | 現行の業務フローとの連携が困難 | 高い |
| **精度への不安** | OCR精度が不十分で手修正が多い | 中-高 |
| **カスタマイズ性** | 自社固有の帳票フォーマットへの対応が難しい | 中-高 |
| **導入期間** | 初期設定・テンプレート作成に時間がかかる | 中 |
| **データセキュリティ** | 帳票データの外部送信に対する懸念 | 中 |

### 1.3 市場トレンド

1. **VLM（Vision Language Model）の台頭**: GPT-4V、Gemini Pro Vision等のVLMにより、従来のOCR+後処理のアプローチからエンドツーエンドの文書理解へ移行が進行中
2. **オンプレミス/ローカルAI回帰**: データセキュリティの観点から、OllamaやvLLM等によるローカル推論の需要が増加
3. **ノーコード/ローコード化**: 非エンジニアでも帳票テンプレートを定義できるUIの需要が増加
4. **電子帳簿保存法対応**: 法令対応を起点としたOCR導入の流れが継続中
5. **API連携の標準化**: 会計ソフト・ERPとのAPI連携が必須要件に

## 3. 差別化ポイント分析

### 3.1 コア差別化要素

#### (1) VLM（Vision Language Model）活用 -- 差別化度: 極めて高い
- **現状**: 競合の大半は従来型OCR + ルールベース後処理。VLMを中核に据えた製品はまだ少ない
- **優位性**: VLMは帳票の「意味」を理解できるため、フォーマット変更やレイアウト差異に対する耐性が高い
- **訴求ポイント**: 「テンプレートを都度修正する手間から解放」「新しい帳票でもすぐに対応」

#### (2) オープンソースモデル（Ollama）対応 -- 差別化度: 高い
- **現状**: 競合はクラウドAPIのみの提供が多く、データ外部送信が必須
- **優位性**: ローカル実行により (a) API費用ゼロ (b) データがサーバー外に出ない (c) インターネット不要での運用が可能
- **訴求ポイント**: 「機密性の高い帳票も安心」「API費用の心配なし」「オフライン環境でも動作」

#### (3) プロンプトによる柔軟な抽出指示 -- 差別化度: 高い
- **現状**: 競合はGUI上の設定項目に限定。抽出ロジックのカスタマイズは開発者対応
- **優位性**: 自然言語プロンプトで抽出指示を記述できるため、非エンジニアでもカスタマイズ可能
- **訴求ポイント**: 「日本語で指示するだけで抽出ルールを変更」「列ごとの変換ルールもプロンプトで」

#### (4) バッチ処理 + 自動監視（Auto-Watch） -- 差別化度: 中-高
- **現状**: バッチ処理対応の競合はあるが、フォルダ自動監視 + Webhook通知まで含む製品は少ない
- **優位性**: 「置いたら処理される」完全自動化ワークフローが構築可能
- **訴求ポイント**: 「フォルダにPDFを入れるだけ」「処理完了をSlack/メールで通知」

## 4. 製品ポジショニング

### 4.1 ターゲットセグメント

#### プライマリターゲット
- **中小企業（従業員10-100名）の事務・経理部門**
  - 月間処理帳票: 100-1,000枚程度

  - ペイン: 人手不足、入力ミス、残業の原因

  - 予算: 月額3-10万円程度

#### ターシャリターゲット
- **中堅企業（従業員100-500名）のDX推進部門**

## 5. SWOT分析

### Strengths（強み）
- VLM技術を活用した次世代型OCR（競合にない技術優位）
- ローカルAI（Ollama）対応によるコスト優位性とセキュリティ担保
- プロンプトベースの柔軟なカスタマイズ性
- 4つの読取パターンによる幅広い帳票対応
- 自動監視（Auto-Watch）による完全自動化ワークフロー
- 中小企業向けに特化した手頃な価格設定

### Weaknesses（弱み）
- 自社プロダクトのため、導入実績がゼロの状態からのスタート
- ブランド認知度がない
- VLM精度は使用モデルに依存（Ollamaのローカルモデルは精度面で課題の可能性）
- サポート体制の構築が必要
- PDFドラッグ範囲指定UIの技術的リスク

### Opportunities（機会）
- 電子帳簿保存法・インボイス制度対応の需要継続
- VLM/LLM技術の急速な進歩（精度向上が見込める）
- 中小企業のDX遅延による潜在市場の大きさ
- 競合の多くがVLM未対応（先行者優位を獲得可能）
- オープンソースAIモデルの充実

### Threats（脅威）
- 大手（Google, Amazon, Microsoft）のAI-OCR機能強化
- 競合のVLM対応の追随
- AI-OCR市場の価格競争激化
- 技術の陳腐化スピード
- 中小企業のDX投資意欲の変動
