# 自社ホームページ - 案件概要

## 基本情報
- **案件ID**: COMPANY-WEBSITE
- **案件名**: 自社ホームページ開発・運営
- **種別**: 常設案件（継続的な開発・運営）
- **Phase**: 企画

## 概要
Webアプリ開発組織としての自社ホームページを開発・運営する。
実績ポートフォリオ、サービス紹介、お問い合わせフォームを備え、
新規案件獲得とブランド構築を目的とする。

## 目的
1. 組織の信頼性・専門性をアピール
2. 実績ポートフォリオによる営業支援
3. お問い合わせからの新規案件獲得
4. 技術ブログによるSEO集客・ブランディング

## HP構成（計画）

### ページ構成
1. **トップページ**
   - ヒーローセクション（キャッチコピー + CTA）
   - サービス概要
   - 実績ハイライト（3-4件）
   - お客様の声（実績が増えてから）
   - CTAセクション

2. **サービス紹介** (`/services`)
   - 提供サービスの詳細
   - 対応技術スタック
   - 開発プロセスの紹介
   - 料金目安

3. **実績・ポートフォリオ** (`/works`)
   - 完了案件の一覧（カード形式）
   - 各案件の詳細ページ
   - フィルター（技術スタック、業種）

4. **会社概要** (`/about`)
   - ミッション・ビジョン
   - 強み・特徴
   - 開発体制

5. **ブログ** (`/blog`)
   - 技術記事
   - 事例紹介
   - お知らせ

6. **お問い合わせ** (`/contact`)
   - 問い合わせフォーム
   - サービス選択
   - 予算・期間の選択肢

## 技術スタック（予定）
- Next.js (App Router)
- TypeScript
- Tailwind CSS + shadcn/ui
- Vercel（ホスティング）
- Geist（フォント）

## 運営方針
- 案件完了ごとにポートフォリオを更新
- 月1回以上のブログ記事公開を目標
- SEOを意識したコンテンツ設計
- Core Web Vitalsの基準を満たすパフォーマンス
