プリセット、S3連携等を実装し、本番運用に向けた安定性・パフォーマンスを確保する。

**Phase 3 成果物**:
- プリセットテンプレート（請求書/納品書/見積書/注文書/領収書 + カスタム）
- 精度統計ダッシュボード（スキーマ別/フィールド別の精度可視化）

- プロンプト改善半自動化（修正パターン分析からの改善提案）
- パフォーマンスチューニング
- E2Eテスト充実
- セキュリティレビュー
- 本番デプロイ準備

**プリセット定義仕様**:
```
各プリセットは以下の情報を持つ:
- id: プリセット識別子（invoice, delivery_note, quotation, purchase_order, receipt）
- name: 日本語表示名
- description: 説明文
- icon: lucide-reactアイコン名

G-04 注文書: 発注先、発注日、注文番号、明細（品名/品番/数量/単価/金額）、納期
G-05 領収書: 発行元、宛名、金額、日付、但し書き
```

### T-9.2: プリセットAPI

| 項目 | 内容 |
|------|------|
| タスク名 | プリセット一覧取得・プリセットからスキーマ作成API |
| 説明 | GET: プリセットテンプレート一覧を返す。POST: 選択されたプリセットIDに基づいてextraction_schemaとextraction_fieldsを一括作成する。認証必須 |
| 成果物 | - `src/app/api/schemas/presets/route.ts`（GET: プリセット一覧、POST: プリセットからスキーマ+フィールド一括作成） |
| 依存関係 | T-9.1 |
| 推定工数 | 3時間 |
| ステータス | [ ] |

**API仕様**:
```
GET /api/schemas/presets
  Response: { data: PresetTemplate[] }

POST /api/schemas/presets
  Request: { preset_id: string, name?: string, description?: string }
  処理:
    1. preset_idに対応するプリセットテンプレートを取得
    2. extraction_schemasにinsert（名前はプリセット名 or カスタム名）
    3. プリセットのフィールド配列をextraction_fieldsに一括insert
    4. 作成されたスキーマ+フィールドを返す
  Response: { data: ExtractionSchema & { fields: ExtractionField[] } }
```
