# PRJ-012 (Sumi / ccmux-ide-gui) v1.28.0 — LocalServersPanel に Sumi 起動分のみ filter

- 担当: dev 部門
- 完了日: 2026-04-26
- 関連: v1.23.0 / DEC-069 (LocalServersPanel 初版) の延長線。v1.23.0 では OS 上の
  全 LISTEN port を一覧表示していたため、他アプリの dev server や OS 自身の
  listen port (Windows svchost / macOS mDNSResponder 等) が混在し、Sumi 経由で
  起動した dev server を見分けにくいという UX 課題があった。
  v1.28.0 で Sumi の組込ターミナル (pty) で起動した process ツリーに含まれる
  pid のみを既定で表示するように改修。

## 1. Rust `collect_sumi_descendant_pids` 実装概要

`src-tauri/src/commands/local_servers.rs`:

- `list_local_servers` のシグネチャを変更:
  `list_local_servers(pty_state: State<PtyState>, filter_by_sumi_only: bool)`
  (`#[tauri::command(rename_all = "camelCase")]` で frontend 側は
  `{ filterBySumiOnly: boolean }` を渡す)。
- 新 helper `collect_pty_root_pids(pty_state) -> HashSet<u32>`:
  `PtyState::ptys (Mutex<HashMap<pty_id, PtyHandle>>)` を走査し、
  `PtyHandle.child` の `portable_pty::Child::process_id() -> Option<u32>` で
  各 pty の root pid を取得。Child が既に exit している場合は None で skip。
- 新 helper `collect_sumi_descendant_pids(roots: &HashSet<u32>, sys: &System) -> HashSet<u32>`:
  1. roots が空なら即 empty を返す (短絡)
  2. sysinfo の全プロセスを 1 回スキャンし `parent_pid -> children list` の
     逆引き map を build (O(N))
  3. 各 root から BFS で descendant を全列挙 (visited set で cycle 防御)
- `LocalServer` struct に `is_sumi_spawned: bool` field を追加 (camelCase で
  `isSumiSpawned` として frontend に届く)。
- `list_local_servers_impl` 内で `is_self = pid == self_pid` と
  `is_sumi_spawned = !is_self && descendants.contains(&pid)` を計算し、
  `filter_by_sumi_only=true` のときは `is_sumi_spawned=false` の row を skip。
- `cargo test --lib commands::local_servers` で 9 テスト PASS:
  - 既存 4 テスト (host_priority / kill_self_pid x2 / list_returns_ok)
  - v1.28.0 追加 5 テスト:
    - `list_with_sumi_only_filter_and_no_pty_yields_empty`
    - `collect_descendants_empty_roots_yields_empty`
    - `collect_descendants_includes_self_root` (BFS の起点契約)
    - `local_server_serializes_is_sumi_spawned_camel_case`
    - `local_server_is_self_flag_struct_construction` (is_sumi_spawned=false 検証)

注意点:

- sysinfo 0.30 の `Process::parent() -> Option<Pid>` を逆引きに使用。
  繰り返し呼ばずに 1 回だけ refresh + map build (O(N)) で済む。
- pty が空 (terminal を 1 つも開いていない) のときは roots 空 → descendants 空、
  filter_by_sumi_only=true で **必ず空配列**を返す契約。
- 自プロセス pid (Sumi 自身) は `is_self=true` で識別され、`is_sumi_spawned` は
  常に false (二重表示防止 + 既存の kill 候補除外動作を維持)。

## 2. Frontend toggle UI 設計

`components/sidebar/LocalServersPanel.tsx`:

- `useState<boolean>(() => readSumiOnlyPref())` で初期値を localStorage から
  同期 read (再 mount 時の flash 回避)。`SUMI_ONLY_LS_KEY = "sumi:local-servers:sumi-only"`、
  値は `"1"` / `"0"`、未設定 / parse エラー時は **既定 true** (オーナー要望優先)。
- header 右側に toggle button を追加 (`<Filter />` (ON) / `<Globe />` (OFF) icon 切替)。
  ON 時は `text-emerald-600` で視覚的に「絞り込み中」を強調。`aria-pressed` /
  `aria-label` で a11y 対応。
- 切替ハンドラ `handleSumiOnlyToggle`: state + localStorage + 即時 fetch を 1 関数で実行。
  state 反映を待たず最新値を渡すため `fetchServers("manual", overrideSumiOnly)` を
  追加 (closure 鮮度問題は `sumiOnlyRef` で polling callback にも反映)。
- `invoke<LocalServer[]>("list_local_servers", { filterBySumiOnly })` で Rust
  に値を渡す (Tauri は `rename_all = "camelCase"` で snake_case → camelCase 変換)。
- 空状態文言を toggle に応じて切替:
  - ON: 「Sumi が起動したサーバーはありません。ターミナルで `npm run dev` 等を
    実行すると表示されます」
  - OFF: 「稼働中の localhost サーバーはありません」(既存文言)
- 各 row に「Sumi」バッジ (`bg-emerald-500/15 text-emerald-700`) を `is_sumi_spawned=true`
  かつ `is_self=false` のときに表示。「すべて」モードでも 1 目で Sumi 起動分が
  識別できる。
- 絵文字は使用していない (組織ルール厳守)。lucide-react `Filter` / `Globe` icon のみ。

## 3. 実機検証結果

`npm run tauri:dev` で起動した Sumi 上で以下を実機検証:

1. **既定 ON (Sumi 起動分のみ) で空状態確認**: ターミナル 0 個 / Sumi 経由起動
   server 0 個の状態で LocalServersPanel を開くと、「Sumi が起動したサーバーは
   ありません。ターミナルで `npm run dev` 等を実行すると表示されます」が表示される
   ことを確認。
2. **Sumi のターミナルで http-server 起動**: workspace でターミナルを開き、
   `npx http-server -p 8080` を実行。LocalServersPanel に :8080 (node.exe) が
   1 行だけ表示されることを確認。「Sumi」バッジ (緑) も付与。
3. **toggle OFF (すべて)**: header の `Globe` icon をクリックして「すべて」モードに
   切替。:8080 に加えて Windows svchost / OS daemon 等の他 LISTEN port が表示。
   :8080 の row には引き続き「Sumi」バッジが表示され、識別可能であることを確認。
4. **toggle ON 復帰**: `Filter` icon をクリックして「Sumi 起動分のみ」に戻す。
   :8080 のみ表示に戻る。
5. **localStorage persist**: Sumi をリロードしても toggle 状態が保持される
   (DevTools → Application → Local Storage で `sumi:local-servers:sumi-only` の
   値を確認)。
6. **ターミナル kill**: ターミナル tab の close button で http-server を停止。
   5 秒以内 (polling 周期) に :8080 row が panel から消えることを確認。
7. **複数 server**: 別ターミナルで `npx http-server -p 9090` を追加起動 →
   2 行 (8080 / 9090) が表示。両方に「Sumi」バッジ。
8. **Sumi 自身 (Tauri WebView2 listen port 等) の扱い**: 「すべて」モードでも
   Sumi 自身の pid は `is_self=true` で「Sumi 自身」灰バッジ + 停止ボタン
   disabled になることを確認 (既存挙動の維持)。

DEC-069 で報告された v1.23.0 の混在表示問題が解消され、開発者は自分が起動した
dev server だけを panel で操作できるようになった。
